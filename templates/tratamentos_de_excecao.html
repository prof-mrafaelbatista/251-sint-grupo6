Tratamentos de Exceção em Python

Conceito

Tratamento de exceções é um mecanismo em Python (e em muitas outras linguagens de programação) que permite que você lide com erros ou eventos inesperados que ocorrem durante a execução de um programa, sem que ele pare abruptamente. Em vez de "quebrar", o programa pode "capturar" a exceção e executar um código específico para lidar com ela, garantindo uma experiência mais robusta e amigável para o usuário.

Os blocos principais para tratamento de exceções são:

  • try: Onde você coloca o código que pode gerar uma exceção.
  • except: Onde você especifica o tipo de exceção a ser capturada e o código a ser executado caso essa exceção ocorra.
  • else: (Opcional) O código aqui é executado se nenhuma exceção ocorrer no bloco try.
  • finally: (Opcional) O código aqui é sempre executado, ocorrendo ou não uma exceção. É útil para tarefas de limpeza, como fechar arquivos.

Aplicação

O tratamento de exceções é fundamental para criar aplicações mais seguras e confiáveis. Ele é aplicado em diversas situações, como:

  • Validação de entrada de usuário: Para garantir que o usuário insira dados no formato correto (ex: um número quando um número é esperado).
  • Operações de arquivo: Para lidar com casos onde um arquivo não existe, não pode ser lido ou está corrompido.
  • Conexões de rede/banco de dados: Para gerenciar falhas de conexão ou problemas de comunicação.
  • Divisão por zero: Evitar erros matemáticos.
  • Acessos a índices inválidos em listas/arrays: Prevenir erros ao tentar acessar uma posição que não existe.

Exemplo de Código

Considere o exemplo de um programa que pede ao usuário para inserir dois números e realizar uma divisão:

try:
    num1_str = input("Digite o primeiro número: ")
    num2_str = input("Digite o segundo número: ")

    num1 = float(num1_str)
    num2 = float(num2_str)

    resultado = num1 / num2
    print(f"O resultado da divisão é: {resultado}")

except ValueError:
    print("Erro: Entrada inválida. Por favor, digite apenas números.")
except ZeroDivisionError:
    print("Erro: Não é possível dividir por zero.")
except Exception as e:
    print(f"Ocorreu um erro inesperado: {e}")
else:
    print("Divisão realizada com sucesso!")
finally:
    print("Fim do programa de divisão.")

Neste exemplo:

  • O bloco try tenta converter as entradas para números e realizar a divisão.
  • Se a conversão falhar (por exemplo, se o usuário digitar texto), um ValueError é capturado.
  • Se o segundo número for zero, um ZeroDivisionError é capturado.
  • Um Exception genérico captura qualquer outro erro inesperado.
  • O bloco else é executado apenas se nenhuma exceção ocorrer.
  • O bloco finally é sempre executado, independentemente de erro ou não.
